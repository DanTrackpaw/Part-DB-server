Part-DB SQLite Category Export - Modular
Architecture
Overview
A completely isolated, self-contained module for exporting Part-DB inventory as a flat
SQLite database with per-category tables. Minimal mainline impact — can be added
without modifying core Part-DB code.
Modularity Strategy
A. Completely Self-Contained
The entire feature lives in one new directory:
src/Services/ImportExportSystem/SQLiteCategoryExporter/
├── SQLiteCategoryExporter.php# Main orchestrator (Symfony service)
├── CategoryTableBuilder.php# Creates & populates tables
├── PartFieldMapper.php# Maps Part entities → table columns
├── SQLiteConnectionFactory.php# Creates temp SQLite connections
└── Resources/
└── schema_template.sql
B. Zero Mainline Changes Needed
This feature requires NO modifications to:
Controllers
Routes
Entities
Database schema
Doctrine configuration
Services configuration
# Optional: base schema DDLOptional single-point integration (1 tiny touch):
Add one export button to Tools UI (or make it a CLI command only)

2. Category Table Builder (Single Responsibility)
// src/Services/ImportExportSystem/SQLiteCategoryExporter/CategoryTableBuilder.php
class CategoryTableBuilder
{
public function __construct(
private PartFieldMapper $fieldMapper,
) {}
public function createAndPopulateTable(
\PDO $connection,
Category $category
): void {
// Generate safe table name from category
$tableName = $this->sanitizeTableName($category);
// Get all parts in this category (recursive)
$parts = $category->getParts();
if (empty($parts)) {
return; // Skip empty categories
}
// Build CREATE TABLE statement from first part's fields
$columns = $this->fieldMapper->getPartColumns($parts[0]);
$createSql = $this->buildCreateTableSQL($tableName, $columns);
// Execute CREATE TABLE
$connection->exec($createSql);
// Prepare INSERT statement
$insertSql = $this->buildInsertSQL($tableName, $columns);
$stmt = $connection->prepare($insertSql);// Populate table with all parts
foreach ($parts as $part) {
$values = $this->fieldMapper->mapPartToRow($part, $columns);
$stmt->execute($values);
}
}
private function buildCreateTableSQL(string $table, array $columns): string
{
$defs = [];
foreach ($columns as $name => $type) {
$defs[] = "`$name` $type";
}
return "CREATE TABLE `$table` (\n
. implode(",\n
id INTEGER PRIMARY KEY,\n
"
", $defs) . "\n)";
}
}
3. Field Mapper (Extensible)
// src/Services/ImportExportSystem/SQLiteCategoryExporter/PartFieldMapper.php
class PartFieldMapper
{
/**
* Define which Part fields to export and their SQLite types
*/
public function getPartColumns(Part $part): array
{
return [
'name' => 'TEXT NOT NULL',
'ipn' => 'TEXT',
'description' => 'TEXT',
'category' => 'TEXT',
'comment' => 'TEXT',
'created_at' => 'DATETIME',
'updated_at' => 'DATETIME',
'instock' => 'INTEGER DEFAULT 0',
// Add parameters as JSON column for flexibility
'parameters' => 'TEXT', // JSON string
];
}
/**
* Extract data from Part entity
*/public function mapPartToRow(Part $part, array $columns): array
{
$row = [];
foreach (array_keys($columns) as $fieldName) {
$row[$fieldName] = match($fieldName) {
'name' => $part->getName(),
'ipn' => $part->getIPN(),
'instock' => $part->getInstock(),
'parameters' => json_encode($this->paramsToArray($part)),
// ... other mappings
default => null,
};
}
return $row;
}
private function paramsToArray(Part $part): array
{
// Convert Part's parameter objects to simple array
// for JSON serialization
return array_map(fn($p) => [
'name' => $p->getName(),
'value' => $p->getValue(),
'unit' => $p->getUnit(),
], $part->getParameters()->toArray());
}
}
4. SQLite Connection Factory
// src/Services/ImportExportSystem/SQLiteCategoryExporter/SQLiteConnectionFactory.php
class SQLiteConnectionFactory
{
public function createConnection(string $filePath): \PDO
{
$dsn = "sqlite:$filePath";
$pdo = new \PDO($dsn, null, null, [
\PDO::ATTR_ERRMODE => \PDO::ERRMODE_EXCEPTION,
\PDO::ATTR_DEFAULT_FETCH_MODE => \PDO::FETCH_ASSOC,
]);
// Enable foreign keys if needed$pdo->exec('PRAGMA foreign_keys = ON');
return $pdo;
}
}

Potential Concerns & Answers
Q: What if Part-DB adds new fields in the future?
A: PartFieldMapper.getPartColumns() is the only place to update. One file.
Q: What about nested data (parameters, storage locations)?
A: Handled flexibly via JSON columns, or expand CategoryTableBuilder to create child
tables per category.
Q: What if someone wants to export by manufacturer instead of category?
A: Create ManufacturerTableBuilder , reuse everything else. No mainline changes.
Q: Performance for large inventories?
A: Uses streaming inserts. Can add pagination/batching in CategoryTableBuilder if
needed.
